# 📘 คู่มือไกด์ไลน์การทดสอบ UAT แบบละเอียด (UAT Step-by-Step Testing Guide)

เอกสารฉบับนี้จัดทำขึ้นเพื่อไกด์ไลน์ขั้นตอนการทดสอบระบบ **ComHub (MVP)** ด้วยตนเองอย่างละเอียดทีละขั้นตอน (Step-by-Step) เพื่อช่วยให้คุณสามารถดำเนินการทดสอบ UAT และบันทึกผลลงใน [UAT_REPORT_TEMPLATE.md](./UAT_REPORT_TEMPLATE.md) ได้อย่างถูกต้อง แม่นยำ และครอบคลุมทุกจุดสำคัญ

---

## 🚀 การเตรียมระบบก่อนเริ่มทดสอบ (Pre-requisites)

1. **เปิดเซิร์ฟเวอร์ Backend:**
   ```bash
   cd backend
   npm run dev       # รันที่ http://localhost:3000
   ```
2. **เปิดเซิร์ฟเวอร์ Frontend:**
   ```bash
   cd FrontEnd
   npm run dev       # รันที่ http://localhost:5173
   ```
3. **เตรียมบัญชีสำหรับทดสอบ:**
   - **Customer:** สามารถสมัครใหม่ผ่านหน้าเว็บ หรือใช้อีเมลสมัครทดสอบ
   - **Admin:** อีเมลที่สร้างไว้ล่วงหน้าจาก `seed.sql` หรือสร้างใหม่ผ่าน DB/Script

---

## 👤 1. คู่มือทดสอบ Persona : Customer (ลูกค้า)

---

### 🔹 UAT-C01: สมัครสมาชิก (Register)
- **วัตถุประสงค์:** ตรวจสอบการสมัครสมาชิกบัญชีลูกค้าใหม่
- **ขั้นตอนการทดสอบ (Steps):**
  1. เข้าหน้าแรกของเว็บแล้วกดปุ่ม **"Register"** หรือไปที่ `/auth`
  2. กรอกข้อมูล:
     - **Email:** `testcustomer@comhub.com`
     - **Password:** `Password123` (ต้องอย่างน้อย 8 ตัวอักษร)
     - **First Name:** `Somchai`
     - **Last Name:** `Jaidee`
  3. กดปุ่ม **"Sign Up"**
- **จุดที่ต้องสังเกต (Expected Behavior):**
  - ระบบสมัครสมาชิกสำเร็จ และล็อกอินให้อัตโนมัติ (หรือนำทางไปหน้าหลัก)
  - แถบเมนูด้านบนแสดงชื่อ `Somchai` และมี Token ถูกจัดเก็บใน `localStorage.getItem('comhub_token')`

---

### 🔹 UAT-C02: เข้าสู่ระบบ (Login)
- **วัตถุประสงค์:** ตรวจสอบการยืนยันตัวตนเข้าสู่ระบบด้วย JWT
- **ขั้นตอนการทดสอบ (Steps):**
  1. หากล็อกอินอยู่ ให้กดปุ่ม **"Logout"**
  2. ไปหน้า **"Login"** กรอก Email: `testcustomer@comhub.com` และ Password: `Password123`
  3. กดปุ่ม **"Sign In"**
- **จุดที่ต้องสังเกต (Expected Behavior):**
  - ล็อกอินสำเร็จ เข้าสู่หน้าหลักโดยไม่มี Error
  - ทดสอบกด Refresh หน้าเว็บ ($F5$) บัญชียังคงสถานะ Login อยู่ ไม่เด้งหลุด

---

### 🔹 UAT-C03: ค้นหาและดูรายละเอียดสินค้า (Product Search & Catalog)
- **วัตถุประสงค์:** ตรวจสอบระบบค้นหา กรองหมวดหมู่ และการแสดงรายละเอียดสเปค
- **ขั้นตอนการทดสอบ (Steps):**
  1. เข้าหน้า **"Catalog"** (`/catalog`)
  2. พิมพ์คำค้นหาในช่อง Search เช่น `"Ryzen"` หรือ `"Intel"`
  3. คลิกเลือกหมวดหมู่ด้านข้าง เช่น `"CPU"` หรือ `"GPU"`
  4. คลิกที่การ์ดสินค้าชิ้นใดชิ้นหนึ่งเพื่อเข้าสู่หน้า **Product Detail**
- **จุดที่ต้องสังเกต (Expected Behavior):**
  - ผลการค้นหาและหมวดหมู่กรองสินค้าได้ถูกต้องและรวดเร็ว ($< 500\text{ms}$)
  - หน้า Product Detail แสดงชื่อ, ราคา, สเปคเทคนิคจาก JSONB (เช่น Socket, TDP, Form Factor) และแสดง **Average Rating** ดาวถูกต้อง

---

### 🔹 UAT-C04: จัดสเปคคอมพิวเตอร์ 7 หมวดหมู่ (PC Builder)
- **วัตถุประสงค์:** ตรวจสอบการเลือกชิ้นส่วนคอมพิวเตอร์ลงใน Bento Grid UI
- **ขั้นตอนการทดสอบ (Steps):**
  1. เข้าหน้า **"PC Builder"** (`/builder`)
  2. คลิกเลือกชิ้นส่วนให้ครบทั้ง **7 หมวดหมู่หลัก**:
     - CPU, Mainboard, GPU, RAM, SSD, Case, PSU
  3. สังเกตกล่องสรุปราคาฝั่งขวา (Summary Panel)
  4. ทดสอบกด Refresh หน้าเว็บ ($F5$)
- **จุดที่ต้องสังเกต (Expected Behavior):**
  - ชิ้นส่วนที่เลือกปรากฏในช่องของหมวดนั้นๆ ราคารวมคำนวณสะสมถูก
  - เมื่อกด Refresh ข้อมูลชิ้นส่วนที่เลือกยังคงอยู่ ไม่หาย (Persist ลง `localStorage`)

---

### 🔹 UAT-C05: ตรวจสอบความเข้ากันได้ (Compatibility Checker)
- **วัตถุประสงค์:** ตรวจสอบการเตือนอุปกรณ์ที่ไม่เข้ากัน (Socket, Case Size, GPU Length, RAM Type)
- **ขั้นตอนการทดสอบ (Steps):**
  1. ในหน้า PC Builder ลองเลือกชิ้นส่วนที่ไม่เข้ากัน เช่น:
     - **CPU:** Intel LGA1700 + **Mainboard:** AMD AM5 (Socket ไม่ตรง)
     - **Mainboard:** ATX + **Case:** Mini-ITX (ขนาด Mainboard ใหญ่เกินเคส)
     - **RAM:** DDR4 + **Mainboard:** รองรับเฉพาะ DDR5 (RAM Type ไม่ตรง)
  2. สังเกตกล่อง **Compatibility Health Panel** ด้านล่าง
- **จุดที่ต้องสังเกต (Expected Behavior):**
  - ระบบแสดงป้ายสถานะสีแดงเตือนข้อผิดพลาดทันที พร้อมระบุสาเหตุชัดเจน เช่น `"Socket mismatch"` หรือ `"RAM Type mismatch"`

---

### 🔹 UAT-C06: คำนวณกำลังไฟและแนะนำ PSU (Wattage Calculator)
- **วัตถุประสงค์:** ตรวจสอบการคำนวณกำลังไฟ TDP รวมและการแนะนำ PSU $\ge 1.2\times\text{TDP}$
- **ขั้นตอนการทดสอบ (Steps):**
  1. เลือก CPU (เช่น TDP 120W) และ GPU (เช่น TDP 220W)
  2. สังเกตค่ากำลังไฟรวมที่ระบบคำนวณ (เช่น 340W)
  3. คลิกเลือกหมวดหมู่ **PSU**
- **จุดที่ต้องสังเกต (Expected Behavior):**
  - ระบบแสดงผลรวม TDP ถูกต้อง
  - ในช่องเลือก PSU ระบบกรองและแนะนำเฉพาะ PSU ที่มีขนาดกำลังไฟเพียงพอ ($\ge 340\text{W} \times 1.2 = 408\text{W}$ ขึ้นไป)

---

### 🔹 UAT-C07: เปรียบเทียบสินค้า (Product Comparison)
- **วัตถุประสงค์:** ตรวจสอบการเปรียบเทียบสเปคสินค้าประเภทเดียวกัน Side-by-Side
- **ขั้นตอนการทดสอบ (Steps):**
  1. ในหน้า Catalog หรือ Product Detail กดปุ่ม **"Add to Compare"** บน CPU 2-3 ชิ้น
  2. เปิดหน้า **Product Comparison** (`/compare`)
  3. ทดสอบกดลบสินค้าออกจากตารางเปรียบเทียบ 1 ชิ้น
- **จุดที่ต้องสังเกต (Expected Behavior):**
  - ระบบแสดงตารางเปรียบเทียบคุณสมบัติเชิงเทคนิค (Socket, Cores, Speed, TDP) เคียงข้างกันชัดเจน
  - ลบสินค้าออกจากตารางได้ถูกต้อง

---

### 🔹 UAT-C08: จัดการรายการโปรด (Wishlist)
- **วัตถุประสงค์:** ตรวจสอบการเพิ่ม/ลบสินค้าใน Wishlist และการเปิด/ปิดแจ้งเตือน
- **ขั้นตอนการทดสอบ (Steps):**
  1. คลิกไอคอนหัวใจบนสินค้าที่ชอบ
  2. เปิดหน้า **Wishlist** (`/wishlist`)
  3. ทดสอบสลับปุ่มสวิตช์ **Stock Alert Toggle**
  4. กดปุ่มลบสินค้าออกจาก Wishlist
- **จุดที่ต้องสังเกต (Expected Behavior):**
  - สินค้าเพิ่มเข้า Wishlist และแสดงผลในหน้า Wishlist ถูกต้อง
  - การสลับสวิตช์ Alert บันทึกสถานะได้ และปุ่มลบสามารถลบสินค้าออกได้ทันที

---

### 🔹 UAT-C09: เขียนรีวิวสินค้า (Review Submission)
- **วัตถุประสงค์:** ตรวจสอบการให้คะแนนดาวและรีวิวข้อความในหน้าสินค้า
- **ขั้นตอนการทดสอบ (Steps):**
  1. เข้าหน้า Product Detail ของสินค้าที่เคยสั่งซื้อ
  2. เลื่อนลงมาส่วน Review Form ให้คะแนน 5 ดาว และพิมพ์ข้อความ `"สินค้าใช้งานได้ดีมาก จัดส่งไว"`
  3. กดปุ่ม **"Submit Review"**
- **จุดที่ต้องสังเกต (Expected Behavior):**
  - รีวิวใหม่ปรากฏในรายการรีวิวด้านล่างทันที
  - ค่า **Average Rating** ของสินค้าได้รับการคำนวณอัปเดตใหม่
  - หากทดสอบส่งรีวิวซ้ำในสินค้าเดิม ระบบจะแจ้งเตือนว่าเคยรีวิวแล้ว (409 Duplicate)

---

### 🔹 UAT-C10: ตะกร้าสินค้า (Cart Management)
- **วัตถุประสงค์:** ตรวจสอบการเลือก/ปรับจำนวน/ลบสินค้าในตะกร้า (Shopee-style)
- **ขั้นตอนการทดสอบ (Steps):**
  1. เพิ่มสินค้าลงตะกร้า 2-3 ชิ้น
  2. เปิดหน้าตะกร้าสินค้า (`/cart`)
  3. ลองติ๊ก Checkbox เลือกสินค้าบางรายการ / ติ๊ก "เลือกทั้งหมด"
  4. ปรับจำนวนสินค้าเพิ่ม-ลด
- **จุดที่ต้องสังเกต (Expected Behavior):**
  - ราคารวมสรุปเฉพาะสินค้าที่ติ๊ก Checkbox เลือกเท่านั้น
  - ปรับจำนวนสินค้า ยอดรวมคำนวณใหม่ถูกต้องแม่นยำ

---

### 🔹 UAT-C11: สั่งซื้อและอัปโหลดสลิป (Checkout & WebP Upload)
- **วัตถุประสงค์:** ตรวจสอบการกรอกที่อยู่ อัปโหลดสลิป และบีบอัดภาพ WebP ฝั่ง Client
- **ขั้นตอนการทดสอบ (Steps):**
  1. จากหน้าตะกร้า กดปุ่ม **"Proceed to Checkout"**
  2. กรอกที่อยู่จัดส่งให้ครบถ้วน
  3. เลือกไฟล์รูปภาพสลิปโอนเงิน (JPG หรือ PNG ขนาด > 1MB)
  4. กดปุ่ม **"Confirm & Place Order"**
- **จุดที่ต้องสังเกต (Expected Behavior):**
  - รูปสลิปถูกบีบอัดผ่าน Canvas API เป็น WebP 80% (ขนาดไฟล์ลดลงเหลือ $<100\text{KB}$)
  - สร้างออเดอร์สำเร็จ สถานะเริ่มต้นเป็น `Pending Payment` และตัดสต็อกสินค้าในฐานข้อมูลทันที

---

### 🔹 UAT-C12: ติดตามสถานะออเดอร์ (Order Tracking Timeline)
- **วัตถุประสงค์:** ตรวจสอบหน้าติดตามออเดอร์และไทม์ไลน์ประวัติสถานะ
- **ขั้นตอนการทดสอบ (Steps):**
  1. เข้าหน้า **"Order Tracking"** (`/order-tracking`) หรือดูออเดอร์ล่าสุด
  2. คลิกเปิดดูรายละเอียดออเดอร์
- **จุดที่ต้องสังเกต (Expected Behavior):**
  - แสดงสถานะออเดอร์ 5 ขั้นตอน (`Pending Payment` $\rightarrow$ `Paid` $\rightarrow$ `Processing` $\rightarrow$ `Shipped` $\rightarrow$ `Delivered`)
  - มีไทม์ไลน์บันทึกประวัติวันเวลาจาก `order_logs` และแสดงเลข Tracking Number เมื่อสินค้าถูกจัดส่งแล้ว

---

## 🔐 2. คู่มือทดสอบ Persona : Admin (ผู้ดูแลระบบ)

---

### 🔹 UAT-A01: เข้าสู่ระบบแอดมินและการป้องกันสิทธิ์ (Admin Login & Guard)
- **ขั้นตอนการทดสอบ:**
  1. ล็อกอินด้วยบัญชี Admin
  2. เข้าหน้า **Admin Dashboard** (`/admin-dashboard`)
  3. ลองใช้บัญชี Customer ทั่วไปพยายามพิมพ์ URL เข้าหน้า `/admin-dashboard`
- **จุดที่ต้องสังเกต:**
  - บัญชี Admin เข้าสู่เมนูแอดมินได้เรียบร้อย
  - บัญชี Customer ถูกระบบบล็อกด้วย Route Guard (ขึ้น 403 Forbidden หรือดีดกลับหน้าหลัก)

---

### 🔹 UAT-A02: จัดการคลังสินค้า (Product CRUD & Soft Delete)
- **ขั้นตอนการทดสอบ:**
  1. เข้าหน้า **Admin Products** (`/admin-products`)
  2. กด **"Add Product"** เพิ่มสินค้าใหม่
  3. แก้ไขราคาสินค้า
  4. กดสวิตช์ **Toggle Status** ปิดการขายสินค้า (Soft Delete `is_active = false`)
- **จุดที่ต้องสังเกต:**
  - สินค้าเพิ่มและแก้ไขได้สำเร็จ
  - สินค้าที่ถูกซ่อน (`is_active = false`) จะไม่ปรากฏในหน้า Catalog ของลูกค้า แต่ยังคงอยู่ใน Admin Panel

---

### 🔹 UAT-A03: ตรวจสอบและอนุมัติสลิปโอนเงิน (Payment Review & Rollback)
- **ขั้นตอนการทดสอบ:**
  1. เข้าหน้า **Payment Review** (`/admin-payment`)
  2. คลิกดูรูปสลิปโอนเงินที่ลูกค้าอัปโหลดเข้ามา
  3. ลองกดปุ่ม **"Approve"** ในออเดอร์ที่ 1 (สถานะเปลี่ยนเป็น `Paid`)
  4. ลองกดปุ่ม **"Reject"** ในออเดอร์ที่ 2 (ระบุเหตุผล)
- **จุดที่ต้องสังเกต:**
  - เมื่อกด Approve ออเดอร์เปลี่ยนสถานะเป็น `Paid`
  - เมื่อกด Reject สต็อกสินค้าในออเดอร์นั้นจะถูกคืนกลับเข้าคลังสินค้าอัตโนมัติ (Stock Rollback)

---

### 🔹 UAT-A04: จัดการออเดอร์และกรอก Tracking Number (Order Management)
- **ขั้นตอนการทดสอบ:**
  1. เข้าหน้า **Order Management** (`/admin-orders`)
  2. เปลี่ยนสถานะออเดอร์จาก `Paid` $\rightarrow$ `Processing` $\rightarrow$ `Shipped`
  3. กรอกหมายเลขพัสดุ (Tracking Number) เช่น `TH123456789`
- **จุดที่ต้องสังเกต:**
  - สถานะออเดอร์อัปเดตถูกต้อง และบันทึกประทับเวลาลง `order_logs`
  - เลขพัสดุที่กรอกแสดงผลในหน้า Order Tracking ฝั่งลูกค้าทันที

---

### 🔹 UAT-A05: ดู Dashboard รายงานยอดขายและเตือนสต็อกต่ำ (Admin Dashboard)
- **ขั้นตอนการทดสอบ:**
  1. เข้าหน้า **Admin Dashboard** (`/admin-dashboard`)
  2. ตรวจสอบการแสดงผลการ์ดสรุปยอดขายรวม, ตารางสินค้ายอดนิยม Top 5, และตารางเตือนสต็อกต่ำ
- **จุดที่ต้องสังเกต:**
  - ยอดขายรวมคำนวณจากออเดอร์ที่จัดส่งสำเร็จแล้ว
  - ตารางสต็อกต่ำแสดงเฉพาะสินค้าที่มีจำนวนสต็อก $\le 3$ ชิ้นพร้อม Highlight สีเตือน

---

### 🔹 UAT-A06: จัดการสิทธิ์และบัญชีผู้ใช้ (Role & Access Control)
- **ขั้นตอนการทดสอบ:**
  1. เข้าหน้า **Admin Accounts** (`/admin-accounts`)
  2. ลองค้นหาชื่อหรืออีเมลผู้ใช้
  3. เปลี่ยนสิทธิ์ผู้ใช้จาก `Customer` $\rightarrow$ `Admin`
- **จุดที่ต้องสังเกต:**
  - ค้นหาผู้ใช้ได้แม่นยำ
  - เมื่อเปลี่ยนสิทธิ์ บัญชีนั้นจะได้สิทธิ์เข้าถึง Admin Panel ในการล็อกอินครั้งถัดไป

---

## ❌ 3. คู่มือทดสอบกรณีเกิดข้อผิดพลาด (Negative & Security Tests)

| รหัสทดสอบ | สิ่งที่ต้องทดสอบ | ขั้นตอนการทดสอบ | ผลลัพธ์ที่ต้องปรากฏ |
| :---: | :--- | :--- | :--- |
| **UAT-N01** | รหัสผ่านผิด | กรอกอีเมลถูก แต่พิมพ์รหัสผ่านผิดในหน้า Login | แสดงข้อความเตือน `"Invalid email or password"` (401) |
| **UAT-N02** | อีเมลซ้ำ | สมัครสมาชิกใหม่โดยใช้อีเมลที่มีในระบบแล้ว | แสดงข้อความเตือน `"Email already registered"` (409) |
| **UAT-N03** | บล็อกสิทธิ์ | ใช้ Customer Token เรียก API `POST /api/products` ผ่าน Postman/Curl | ตอบกลับ `403 Forbidden` ป้องกันการลักลอบบันทึกข้อมูล |
| **UAT-N04** | Rate Limiting | ลองส่ง Login สุ่มรหัสผ่านผิดรัวๆ 6 ครั้งติดต่อกัน | ครั้งที่ 6 ระบบบล็อกตอบกลับ `429 Too Many Requests` ให้รอ 15 นาที |
