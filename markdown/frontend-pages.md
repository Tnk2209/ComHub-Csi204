# สรุปโครงร่างหน้าระบบและหน้าจอฝั่งหน้าบ้าน (Frontend Page List & Specifications) - MVP Version

เอกสารนี้รวบรวมรายละเอียดโครงสร้างหน้าจอแสดงผลหน้าบ้าน (Frontend Pages) ของระบบ **ComHub** แยกตามบทบาทของผู้ใช้งาน 2 บทบาทหลัก (Customer และ Admin) เพื่อใช้เป็นคู่มืออ้างอิงในการออกแบบส่วนติดต่อผู้ใช้งาน (UI/UX Mockups) และเขียนโค้ดหน้าบ้าน

> **📝 หมายเหตุ MVP Scope:** ตัดหน้า Staff และ Manager ออก เน้นเฉพาะ Customer และ Admin เท่านั้น

---

## 🎨 1. บทบาท: ลูกค้า (Customer Pages)

ลูกค้าทั่วไปของระบบอีคอมเมิร์ซและผู้ใช้งานระบบจัดสเปคคอมพิวเตอร์

### 1.1 หน้าแรกและหน้ารายการสินค้าแค็ตตาล็อก (Home & Product Catalog Page)
*   **วัตถุประสงค์:** หน้าแรกในการนำเสนอแบรนด์ ค้นหาข้อมูลชิ้นส่วนไอที และปูทางเข้าสู่ระบบ PC Builder
*   **องค์ประกอบหลักบน UI:**
    *   แบนเนอร์โปรโมชั่น
    *   กล่องค้นหา (Search Box) และตัวเลือกสับประเภทสินค้า 7 หมวด (CPU, Mainboard, GPU, RAM, SSD, Case, PSU)
    *   ปุ่มหยิบใส่ตะกร้าทันที (Add to Cart) และปุ่มรายการของโปรดรูปหัวใจ (Wishlist)
    *   แผงเปรียบเทียบคุณสมบัติ (Product Compare Panel) รองรับการดึงข้อมูล 3 อุปกรณ์มาเทียบกัน

### 1.2 หน้ารายละเอียดสินค้าชิ้นเดี่ยว (Product Detail Page)
*   **วัตถุประสงค์:** แสดงข้อมูลเชิงเทคนิคเชิงลึกและเกณฑ์คะแนนความพึงพอใจของสินค้าชิ้นเดียว
*   **องค์ประกอบหลักบน UI:**
    *   รูปภาพและข้อมูลราคา คีย์ Specifications (เช่น ชนิด RAM, ขั้วซ็อกเก็ต)
    *   กล่องแสดงรีวิว ความคิดเห็น (ไม่มีรูปภาพ - MVP)
    *   ปุ่มกดแอดสินค้าเข้า Wishlist

### 1.3 หน้าจอจัดสเปคคอมพิวเตอร์อัจฉริยะ (Advanced PC Builder Workspace)
*   **วัตถุประสงค์:** เครื่องมือหลักในการจัดชุดคอมพิวเตอร์ด้วยการตรวจสเปคและกำลังไฟแบบกึ่งเรียลไทม์
*   **องค์ประกอบหลักบน UI:**
    *   ตารางรายหมวดหมู่ทั้ง 7 อุปกรณ์หลัก พร้อมปุ่มกด "เลือกอุปกรณ์"
    *   ตัวคำนวณกำลังไฟสะสม (TDP Calculator Progress Bar) แสดงระดับวัตต์ และแนะนำเฉพาะ PSU ที่จ่ายไฟเพียงพอพร้อมตัวคูณความปลอดภัย 20%
    *   กล่องตรรกะ Compatibility Checker คอยเตือนสเปคที่เสียบใช้งานร่วมกันไม่ได้
    *   ปุ่ม "สั่งซื้อชุดนี้ (Checkout System)"

### 1.4 หน้าตะกร้าสินค้าและการดำเนินการสั่งซื้อ (Cart & Checkout Page)
*   **วัตถุประสงค์:** หน้าตรวจสอบชิ้นส่วน ยอดราคารวม และแนบสลิปโอนเงินชำระสินค้า (Mockup)
*   **องค์ประกอบหลักบน UI:**
    *   ตารางสรุปราคา ชิ้นส่วน ค่าจัดส่ง
    *   ภาพ QR Code บัญชีการโอน และช่องอัปโหลดรูปถ่ายหลักฐานสลิปเงิน (Mockup - แปลงเป็น Base64 WebP)

### 1.5 หน้าประวัติสั่งซื้อและติดตามสถานะ (Order Tracking Page)
*   **วัตถุประสงค์:** แดชบอร์ดของลูกค้าในการเช็คความเคลื่อนไหวออเดอร์
*   **องค์ประกอบหลักบน UI:**
    *   แถบไทม์ไลน์ภาพ 5 ขั้นตอน: `[Pending Payment] -> [Paid] -> [Processing] -> [Shipped] -> [Delivered]`
    *   เลข Tracking นำส่งพัสดุ

### 1.6 หน้ารวมของโปรด (Wishlist Page)
*   **วัตถุประสงค์:** จัดเก็บชิ้นส่วนที่กดใจไว้ (C-06 - MVP: ไม่มี Stock Alert)
*   **องค์ประกอบหลักบน UI:**
    *   รายการชิ้นส่วนที่บันทึกไว้ (บันทึกใน `wishlist_items`)
    *   ปุ่มลบออกจาก Wishlist และปุ่มเพิ่มลงตะกร้าโดยตรง

### 1.7 หน้าเข้าสู่ระบบและสมัครสมาชิก (Login / Register Page)
*   **วัตถุประสงค์:** จัดการเข้าสู่ระบบและสร้างบัญชี (SYS-01, C-10)
*   **องค์ประกอบหลักบน UI:**
    *   ฟอร์ม Native Login/Register (email + password)
    *   ปุ่ม "Sign in with Google" / "Sign up with Google" (Google OAuth 2.0 flow)
    *   ข้อความ error สำหรับกรณีอีเมลซ้ำ / รหัสผ่านผิด

### 1.8 หน้าเปรียบเทียบสินค้า (Product Comparison Page)
*   **วัตถุประสงค์:** เปรียบเทียบสเปคเชิงเทคนิคของสินค้าประเภทเดียวกัน สูงสุด 3 ชิ้น (C-05)
*   **องค์ประกอบหลักบน UI:**
    *   ตารางแนวนอนเปรียบเทียบสเปคจาก `products.specifications` (JSONB)
    *   ราคาและปุ่ม "เพิ่มลงตะกร้า"
    *   ปุ่มลบสินค้าออกจากรายการเปรียบเทียบ (Client-side / LocalStorage)

## 🔐 2. บทบาท: ผู้ดูแลระบบ (Admin Pages)

ผู้ควบคุมระบบ จัดการสินค้า อนุมัติการชำระเงิน และดู Dashboard

---


### 2.1 หน้าจัดการสินค้า (Admin Products CRUD Panel)
*   **วัตถุประสงค์:** บริหารจัดการรายการสินค้าหลักของร้านค้า
*   **องค์ประกอบหลักบน UI:**
    *   ฟอร์มเพิ่ม/แก้ไข/ลบรายการสินค้า และระบุค่าข้อมูลเทคนิคเขียนลง specifications JSONB
    *   สวิตช์เปิด/ปิด การวางจำหน่ายสินค้าเดี่ยว (`is_active` Soft Delete)
    *   อัปโหลดรูปภาพสินค้า (WebP)

### 2.2 หน้าอนุมัติการชำระเงิน (Payment Review Panel)
*   **วัตถุประสงค์:** ตรวจทานความถูกต้องธุรกรรมทางการเงินของลูกค้า
*   **องค์ประกอบหลักบน UI:**
    *   แสดงภาพสลิปโอนเงิน (Base64 Mockup) ประกอบกับยอดรวมสั่งซื้อของลูกค้า
    *   ปุ่มกดอนุมัติ (`Approved`) หรือปฏิเสธ (`Rejected`)

### 2.3 หน้าจัดการออเดอร์ (Order Management Panel)
*   **วัตถุประสงค์:** อัปเดตสถานะออเดอร์และกรอก Tracking Number
*   **องค์ประกอบหลักบน UI:**
    *   รายการออเดอร์ทั้งหมด
    *   ปุ่มเปลี่ยนสถานะ: Paid → Processing → Shipped → Delivered
    *   ช่องกรอก Tracking Number

### 2.4 หน้า Dashboard (Admin Dashboard - A-05)
*   **วัตถุประสงค์:** ภาพรวมธุรกิจและสถิติ
*   **องค์ประกอบหลักบน UI:**
    *   สรุปยอดขายรวม
    *   รายการสินค้ายอดนิยม
    *   แจ้งเตือนสต็อกต่ำ (≤ 3 ชิ้น)

### 2.5 หน้าจัดการบัญชีและสิทธิ์ (Admin Accounts & Role Panel - A-04)
*   **วัตถุประสงค์:** สร้างและจัดการบัญชี Admin เพิ่มเติม รวมถึงเปลี่ยนสิทธิ์ระหว่าง Customer/Admin
*   **องค์ประกอบหลักบน UI:**
    *   ตารางรายชื่อผู้ใช้งานทั้งหมด (users) พร้อมคอลัมน์ role
    *   ปุ่ม "Add Admin" เพื่อสร้างบัญชี Admin ใหม่
    *   Dropdown สลับ role (`Customer` ↔ `Admin`)
    *   ปุ่มลบบัญชี
