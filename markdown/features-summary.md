# เอกสารสรุปข้อกำหนดความต้องการระบบ ComHub (System Requirements Summary)

เอกสารนี้รวบรวมและจำแนกความต้องการด้านฟังก์ชันการทำงาน (**Functional Requirements**) และความต้องการที่ไม่ใช่ฟังก์ชัน (**Non-Functional Requirements**) ของระบบ **ComHub** ตามที่กำหนดไว้ในคู่มือขอบเขตระบบและเอกสารข้อกำหนด (TOR) เพื่อเป็นแนวทางและพิมพ์เขียวในการพัฒนาและตรวจรับระบบตามหลัก SDLC

---

## 👥 ผู้ใช้งานและสิทธิ์การเข้าถึงระบบ (Actors & Role-Based Access Control)

ระบบแบ่งออกเป็น 4 บทบาทหลัก ซึ่งจะมีสิทธิ์เข้าถึงฟังก์ชันต่างๆ ต่างกันดังนี้:
1. **ลูกค้า (Customer - C):** ผู้ใช้งานทั่วไปที่ต้องการจัดสเปคคอมพิวเตอร์ เลือกซื้ออุปกรณ์ไอที และติดตามสถานะประกอบ
2. **พนักงาน (Staff - S):** ผู้แลหน้างาน จัดประกอบเครื่องคอมพิวเตอร์ ทำการทดสอบ และบันทึกหมายเลขการจัดส่ง
3. **ผู้จัดการ (Manager - M):** ผู้บริหารร้านค้าไอที ดูแลแดชบอร์ดยอดขาย เทมเพลตโปรโมชั่น และอนุมัติรูปถ่ายรีวิว
4. **ผู้ดูแลระบบ (Administrator - A):** ผู้ควบคุมโครงสร้างระบบ จัดการสิทธิ์พนักงาน คลังสินค้า และเงื่อนไข Compatibility หลัก

---

## 📋 1. ความต้องการด้านฟังก์ชันการทำงาน (Functional Requirements)

ตารางด้านล่างแสดงรายการฟังก์ชันทั้งหมดของระบบ และบทบาทผู้ใช้งานที่มีสิทธิ์เข้าถึงหรือปฏิบัติการ (Role Access Matrix)

| รหัสฟังก์ชัน | ฟังก์ชันระบบ (Feature Name) | รายละเอียดการทำงาน (Description) | สิทธิ์ผู้ใช้งานที่มีสิทธิ์ (Role Access) |
| :---: | :--- | :--- | :---: |
| **SYS-01** | ระบบสิทธิ์และการยืนยันตัวตน (Authentication & Auth) | สมัครสมาชิก, ล็อกอินเข้าสู่ระบบ และการตรวจสอบ Token ของ API หลังบ้าน | Customer, Staff, Manager, Admin |
| **SYS-02** | ระบบตะกร้าสินค้า (Cart & Checkout Flow) | การจัดการสินค้าในตะกร้า, กรอกข้อมูลจัดส่ง และการอัปโหลดไฟล์สลิปหลักฐานโอนเงิน | Customer (หน้าบ้าน), Staff (ตรวจสอบหลังบ้าน) |
| **SYS-03** | ระบบย่อรูปภาพอัจฉริยะ (Client-side WebP Compression) | แปลงรูปภาพสลิป/รีวิวสินค้าเป็น WebP และบีบอัดผ่าน HTML5 Canvas ก่อนส่งขึ้นคลาวด์ | Customer (หน้าบ้าน), Manager/Admin (หลังบ้าน) |
| **C-01** | ระบบเลือกอุปกรณ์จัดสเปค (PC Builder Page) | เลือกซื้อชิ้นส่วนคอมพิวเตอร์แยกตามหมวดหมู่หลัก (CPU, Mainboard, GPU, RAM, SSD, PSU, Case) | Customer |
| **C-02** | ตรรกะจับคู่ Socket และขนาด (Compatibility Checker) | กรองและแจ้งเตือนกรณีชิ้นส่วนไม่รองรับ เช่น บอร์ดคนละ Socket กับ CPU, ขนาดเคสสั้นกว่าขนาดการ์ดจอ | Customer (กรองขณะเลือก), Admin (ตั้งค่ากฎ) |
| **C-03** | ระบบคำนวณกำลังไฟ (Wattage Calculator) | คำนวณ TDP รวมของอุปกรณ์ แนะนำและกรองเฉพาะ PSU ที่มีกำลังวัตต์มากกว่าไฟรวม 1.2 เท่า | Customer |
| **C-04** | เซ็ตสเปคแนะนำของร้าน (Pre-built Templates) | แสดงรายชื่อเซ็ตคอมฯ สำเร็จรูป สามารถกดใส่ตะกร้าหรือดึงไปปรับแต่งต่อใน PC Builder | Customer |
| **C-05** | ระบบตารางเปรียบเทียบสเปค (Product Comparison) | เลือกเปรียบเทียบคุณสมบัติทางเทคนิคและราคาของสินค้าประเภทเดียวกันได้สูงสุด 3 ชิ้น | Customer |
| **C-06** | บันทึกสินค้าโปรด & แจ้งเตือนสต็อก (Wishlist & Stock Alert) | บันทึกสินค้าลงรายการโปรด และรับแจ้งเตือนทางระบบเมื่อสินค้าที่เคยหมดมีสต็อกเพิ่ม | Customer |
| **C-07** | ระบบรีวิวพร้อมรูปภาพ (Review with Photos) | เขียนรีวิว ให้คะแนน 1-5 ดาว และอัปโหลดภาพถ่ายประกอบสลิปหรือสินค้าจริง | Customer |
| **C-08** | แกลเลอรี่ประกอบคอมฯ (PC Build Gallery) | หน้ารวมภาพเคสประกอบคอมฯ สเปคและงบประมาณของลูกค้าคนอื่นเพื่อหาแรงบันดาลใจ | Customer (เข้าชม), Manager (ปักหมุด) |
| **C-09** | ระบบติดตามการประกอบ (Assembly Tracking UI) | หน้าติดตามสถานะสั่งซื้ออย่างละเอียด 4 ขั้นตอน: [รับออเดอร์] -> [กำลังประกอบ] -> [เทสระบบ] -> [จัดส่งแล้ว] | Customer (ดู), Staff (อัปเดต) |
| **S-01** | ระบบจัดการประกอบคอมพิวเตอร์ (Build Management) | แสดงรายการคำสั่งซื้อที่เลือกประกอบเครื่อง อัปเดตสถานะให้ลูกค้าเห็นแบบ Real-time | Staff |
| **S-02** | ระบบลงบันทึกการ Burn-in (Burn-in Test Record) | บันทึกรายงานผลการทดสอบความเสถียรและอุณหภูมิของเครื่องประกอบลงระบบก่อนจัดส่ง | Staff |
| **S-03** | ระบบออกใบจัดส่ง & ใส่ Tracking (Logistics) | ออกพิมพ์ใบนำส่ง และพิมพ์หมายเลขพัสดุ (Tracking Number) เพื่อบันทึกแจ้งลูกค้า | Staff |
| **M-01** | จัดการเทมเพลตแนะนำ (Pre-built Management) | เพิ่ม ลบ แก้ไขสเปคแนะนำของร้าน กำหนดป้ายแท็กช่วงงบประมาณต่างๆ | Manager |
| **M-02** | ตรวจสอบรูปภาพแกลเลอรี่ (Gallery Moderation) | ตรวจสอบและอนุมัติรีวิว/ภาพประกอบของลูกค้า ปักหมุดโพสต์โดดเด่นขึ้นหน้าแรก | Manager |
| **M-03** | แดชบอร์ดวิเคราะห์ยอดขาย (Sales Dashboard) | ดูสถิติสรุปยอดขายรวม สินค้าไอทียอดนิยม และระบบเตือนรายการสินค้าใกล้หมดสต็อก | Manager |
| **A-01** | จัดการคลังข้อมูลหลัก (Database Management CRUD) | เพิ่ม ลบ แก้ไข รายชื่อสินค้าและปรับระดับสต็อกสินค้าในคลัง | Admin |
| **A-02** | ตั้งกฎการเช็คเงื่อนไข (Rules Configuration) | กำหนดจับคู่ Socket ของ CPU และ Mainboard รวมถึงขนาดและความยาวสูงสุดที่เคสรับได้ | Admin |
| **A-03** | ตั้งค่าการใช้พลังงาน (Power Allocation Settings) | ระบุกำลังไฟกินวัตต์ (TDP Watts) ให้กับสินค้าไอทีทุกลำดับในระบบ | Admin |
| **A-04** | จัดการผู้ใช้และตำแหน่ง (Role & Access Control) | สร้างและยกเลิกบัญชีใช้งานของ Staff และ Manager พร้อมกำหนดสิทธิ์การเข้าถึงเมนู | Admin |

## ⚙️ 2. ความต้องการที่ไม่ใช่ฟังก์ชันการทำงาน (Non-Functional Requirements)

ข้อกำหนดคุณลักษณะและประสิทธิภาพด้านคุณภาพระบบ ComHub มีดังนี้:

1. **ด้านประสิทธิภาพการประมวลผล (Performance):**
   - การประมวลผลของเงื่อนไข Compatibility Checker และระบบ Wattage Calculator ในหน้าจอจัดสเปคต้องตอบสนองภายในเวลา **500 มิลลิวินาที (Sub-second response)** หลังลูกค้าทำการเลือกอุปกรณ์เพื่อป้องกันความล่าช้าในประสบการณ์ผู้ใช้
2. **ด้านความปลอดภัยและการแยกสิทธิ์ผู้ใช้ (Security & Access Control):**
   - มีระบบความปลอดภัย Role-Based Access Control (RBAC) ควบคุมที่ Backend API:
     - เข้ารหัสความปลอดภัยพาสเวิร์ด (Password Hashing) ก่อนลง PostgreSQL
     - คุ้มกันเส้นทาง Endpoint ด้วย JWT Verification
     - ป้องกันการเข้าถึงเมนูหลังบ้านของพนักงานหากล็อกอินด้วยบทบาทลูกค้าทั่วไป
3. **ด้านความน่าเชื่อถือและการรักษาข้อมูล (Reliability & State Preservation):**
   - ข้อมูลการจัดสเปคที่ยังจัดค้างอยู่และสินค้าในตะกร้าชั่วคราวต้องบันทึกลงใน **LocalStorage** เสมอ เพื่อรักษาความพึงพอใจของลูกค้าเมื่อเน็ตหลุดหรือปิดเว็บ
   - มีการประมวลผลสต็อกและคลังสินค้าอย่างแม่นยำเพื่อป้องกันการขายเกินจำนวนที่มีอยู่จริง (Overselling Prevention)
4. **ด้านการออกแบบส่วนต่อประสานและฟอนต์ (UI/UX & Design Standards):**
   - แสดงผลแบบ **Fullscreen Responsive Layout** (100% width, min-height 100vh) เพื่อให้แสดงผลได้อย่างราบรื่นบนหน้าจอ Desktop (16:9) และ Mobile
   - การจับคู่ฟอนต์คู่ที่เป็นมาตรฐานพรีเมียมของระบบไอที: ฟอนต์ **'IBM Plex Sans Thai'** สำหรับภาษาไทย และฟอนต์ **'Inter'** สำหรับตัวเลขและภาษาอังกฤษ
   - **รหัสสีต้องห้าม (Purple Ban):** ดีไซน์เป็นธีม Dark Mode ที่พรีเมียม เรียบหรู แต่ห้ามมีรหัสสีโทนสีม่วง (Purple/Violet Hex codes) ใน CSS หรือ Assets ใดๆ
5. **ด้านประสิทธิภาพการใช้พื้นที่จัดเก็บคลาวด์ (Cloud Optimization):**
   - รูปภาพสลิปชำระเงินและรูปถ่ายรีวิวที่ผู้ใช้อัปโหลด ต้องถูกย่อขนาดและแปลงฟอร์แมตเป็น **WebP** ด้วยคุณภาพ 80% (0.8 quality) ผ่าน Canvas API ที่ฝั่งหน้าบ้านก่อนส่งขึ้นคลาวด์ เพื่อประหยัดพื้นที่จำกัด Supabase Storage (1 GB)
