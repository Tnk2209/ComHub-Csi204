# เอกสารวิเคราะห์กลุ่มผู้ใช้งานจำลอง (User Persona Analysis) - MVP Version

เอกสารนี้แสดงการวิเคราะห์กลุ่มผู้ใช้งานจำลอง (**User Personas**) ของระบบ **ComHub** เพื่อทำความเข้าใจพฤติกรรม เป้าหมาย ข้อจำกัด (Pain Points) และการออกแบบฟีเจอร์ของระบบให้ตอบโจทย์ผู้ใช้งานจริง ตามหลักการวิเคราะห์ระบบ (System Analysis) ในวิชา **csi 204**

> **📝 หมายเหตุ MVP Scope:** ปรับให้เหลือ **4 Personas** (3 กลุ่มลูกค้า + 1 แอดมิน) ครอบคลุมเพียง 2 บทบาทตาม MVP คือ Customer และ Admin (ตัด Staff และ Manager ออก) พร้อมทั้งเอาฟีเจอร์ที่ถูกตัดออกจาก Persona Mapping (Pre-built Templates, PC Build Gallery, Assembly & Burn-in Testing, Stock Alert, Review Photos, Coupon)

---

## 🎯 รูปแบบโครงสร้างการวิเคราะห์ User Persona (Persona Template)

ในการจัดทำเล่มรายงาน ข้อมูลของ Persona แต่ละคนจะประกอบด้วยหัวข้อหลักดังนี้:

1. **ประวัติและข้อมูลพื้นฐาน (Demographics & Bio):** ชื่อ, อายุ, อาชีพ, และความคุ้นเคยกับเทคโนโลยี (Tech Savviness)
2. **เป้าหมายและความต้องการ (Goals & Needs):** สิ่งที่คาดหวังจะได้รับจากการเข้าใช้งานระบบ ComHub
3. **ปัญหาและข้อจำกัดในปัจจุบัน (Pain Points & Frustrations):** อุปสรรคที่เจอเมื่อใช้บริการเว็บจัดสเปคคอมฯ หรือเว็บไอทีอื่นๆ
4. **พฤติกรรมการใช้งาน (Behaviors):** ลักษณะการเลือกซื้อสินค้าและการใช้งานอุปกรณ์สื่อสาร
5. **ฟีเจอร์ของ ComHub ที่ตอบโจทย์ (ComHub Mapping):** ฟังก์ชันของระบบที่จะช่วยแก้ปัญหาให้เขาได้โดยตรง (อ้างอิงรหัสฟังก์ชันจาก `project-scope.md`)

## 👥 รายละเอียดกลุ่มผู้ใช้งานจำลอง (Detailed Personas)

### Persona 1: ลูกค้ากลุ่มวัยรุ่น / เกมเมอร์งบจำกัด (The Budget Gamer / Student)

```
[ รูปภาพจำลอง: เด็กหนุ่มวัยรุ่นสวมหูฟังเกมมิ่ง ]
ชื่อ: นายกฤษดา (น้องก้อง) | อายุ: 20 ปี
อาชีพ: นักศึกษาคณะวิศวกรรมศาสตร์
ความคุ้นเคยไอที: ระดับกลาง (ชอบเล่นเกม แต่ไม่เคยประกอบคอมพิวเตอร์ด้วยตัวเอง)
```

* **เป้าหมายและความต้องการ (Goals & Needs):**
  - ต้องการจัดสเปคคอมพิวเตอร์เล่นเกมที่มีงบจำกัดไม่เกิน 25,000 บาท
  - ต้องการเครื่องที่สามารถเล่นเกมระดับ AAA ในปัจจุบันได้ลื่นไหล
  - อยากลองจัดสเปคด้วยตนเองแทนการซื้อแบรนด์สำเร็จรูปเพื่อความคุ้มค่า
  - อยากบันทึกอุปกรณ์ที่สนใจไว้เปรียบเทียบราคาช่วงลดราคา
* **ปัญหาและข้อจำกัด (Pain Points):**
  - ไม่รู้วิธีการตรวจสอบว่าบอร์ดและ CPU ที่เลือกเข้ากันได้ไหม (กลัวซื้อมาแล้วใส่ไม่ได้)
  - ไม่รู้วิธีคำนวณกำลังไฟวัตต์ (TDP) ของ PSU กลัวเครื่องไฟดับหรือพังขณะเล่นเกม
  - มีงบน้อย กลัวโดนร้านค้าหลอกขายสินค้าที่ไม่คุ้มราคา
  - จำรุ่นอุปกรณ์ที่เคยดูไว้ไม่ได้ ต้องเข้ามาค้นหาซ้ำทุกครั้ง
* **พฤติกรรม (Behaviors):**
  - ใช้สมาร์ทโฟนเป็นหลักในการดูข้อมูลสินค้า
  - ชอบอ่านรีวิวและคะแนนของคนที่เคยใช้จริงก่อนตัดสินใจ
  - ใช้ Google Login เพื่อประหยัดเวลากรอกฟอร์ม
* **ฟีเจอร์ ComHub ที่เข้ามาแก้ปัญหา (ComHub Mapping):**
  - **PC Builder & Compatibility Logic (C-01, C-02):** ระบบกรอง Socket และขนาดการ์ดจอบังคับให้เลือกเฉพาะของที่เข้ากันได้
  - **Wattage Calculator (C-03):** มีระบบคำนวณ TDP รวมและกรองเฉพาะ PSU วัตต์ผ่านเกณฑ์ความปลอดภัย 1.2 เท่า
  - **Review System (C-07):** อ่านคะแนนและความคิดเห็น 1-5 ดาวจากลูกค้าที่ซื้อจริง (ไม่มีภาพประกอบ)
  - **Wishlist (C-06):** ปักหมุดชิ้นส่วนที่ถูกใจไว้ในบัญชี เข้ามาดูได้ทุกอุปกรณ์
  - **Google OAuth (C-10):** สมัคร/ล็อกอินด้วย Google Account ในคลิกเดียว

### Persona 2: ลูกค้ากลุ่มคนทำงานสายครีเอทีฟ (The Professional Workstation Creator)

```
[ รูปภาพจำลอง: หญิงสาววัยทำงานนั่งหน้าจอคู่ในสตูดิโอ ]
ชื่อ: นางสาวมนัสวี (คุณเมย์) | อายุ: 28 ปี
อาชีพ: ดีไซเนอร์และนักตัดต่อวิดีโอฟรีแลนซ์
ความคุ้นเคยไอที: ระดับสูง (รู้สเปคคอมพิวเตอร์เป็นอย่างดี ต้องการอุปกรณ์เฉพาะทาง)
```

* **เป้าหมายและความต้องการ (Goals & Needs):**
  - ต้องการประกอบคอมพิวเตอร์สเปคสูง (Workstation) สำหรับตัดต่อวิดีโอ 4K และเรนเดอร์กราฟิก 3D
  - ต้องการเลือกการ์ดจอค่ายเขียวความจุสูง และเมนบอร์ดที่มีพอร์ตเชื่อมต่อครบถ้วน
  - ต้องการเปรียบเทียบสเปคอุปกรณ์รุ่นต่างๆ อย่างละเอียดก่อนตัดสินใจ
* **ปัญหาและข้อจำกัด (Pain Points):**
  - การ์ดจอตัดต่อมักมีขนาดใหญ่และยาวมาก กลัวเลือกเคสมาแล้วใส่การ์ดจอลงไปไม่ได้
  - เจอ RAM ผิดชนิด (DDR4/DDR5) หรือเมนบอร์ดผิดฟอร์มแฟกเตอร์ (ATX/mATX) หลังสั่งซื้อไปแล้ว
  - ทำงานฟรีแลนซ์ ต้องการเห็นสถานะการจัดส่งชัดเจนโปร่งใสตลอดเส้นทาง
* **พฤติกรรม (Behaviors):**
  - ใช้งานคอมพิวเตอร์เดสก์ท็อปเป็นหลักในการเลือกซื้อ
  - ชอบเปรียบเทียบสเปคทางเทคนิคเชิงลึก (เช่น แบนด์วิดท์, บัสแรม, กำลังไฟ)
  - เขียนรีวิวหลังใช้จริงเพื่อช่วยชุมชนตัดสินใจ
* **ฟีเจอร์ ComHub ที่เข้ามาแก้ปัญหา (ComHub Mapping):**
  - **GPU Length & Case Compatibility Check (C-02):** บังคับเช็คขนาดการ์ดจอกับพื้นที่ความกว้างสูงสุดที่เคสรองรับ รวมถึงชนิด RAM (DDR4/DDR5) และฟอร์มแฟกเตอร์ของเมนบอร์ด
  - **Product Comparison Table (C-05):** หน้าเปรียบเทียบสเปคชิ้นส่วนเทียบได้สูงสุด 3 อุปกรณ์
  - **Order Tracking (C-09):** ติดตามสถานะ 5 ขั้น (Pending Payment → Paid → Processing → Shipped → Delivered) พร้อมล็อกเวลาแต่ละขั้นตอนและเลขพัสดุ
  - **Review System (C-07):** เขียนรีวิว 1-5 ดาวพร้อมข้อความ (ไม่มีการอัปโหลดรูปประกอบใน MVP)

### Persona 3: ลูกค้าทั่วไป / ผู้ปกครองจัดคอมฯ ให้ลูก (The Casual Buyer / Parent)

```
[ รูปภาพจำลอง: ชายวัยกลางคนยืนยิ้มในร้านกาแฟ ]
ชื่อ: นายสมเกียรติ (ลุงสมเกียรติ) | อายุ: 52 ปี
อาชีพ: เจ้าของธุรกิจส่วนตัว (ร้านอาหาร)
ความคุ้นเคยไอที: ระดับต่ำ (ใช้งานสมาร์ทโฟนทั่วไปได้ ไม่มีความรู้เรื่องสเปคฮาร์ดแวร์เลย)
```

* **เป้าหมายและความต้องการ (Goals & Needs):**
  - ต้องการซื้อคอมพิวเตอร์ให้ลูกสาวใช้เรียนออนไลน์และทำงานส่งครูในงบกลางๆ
  - ต้องการวิธีสั่งซื้อและชำระเงินที่เข้าใจง่าย (โอนธนาคาร + อัปโหลดสลิป)
  - ต้องการดูสถานะออเดอร์อยู่ตลอดเพื่อความสบายใจ
* **ปัญหาและข้อจำกัด (Pain Points):**
  - สับสนกับศัพท์เทคนิคไอที (เช่น LGA1700, DDR5, PCIe, NVMe, TDP)
  - กลัวจ่ายเงินแล้วไม่รู้ว่าออเดอร์ไปถึงไหน
  - กลัวว่าอุปกรณ์ที่เลือกจะใส่ไม่เข้ากัน (เช่น CPU ผิด Socket)
* **พฤติกรรม (Behaviors):**
  - ซื้อของออนไลน์ผ่านสมาร์ทโฟน
  - เชื่อถือคำแนะนำและระบบเตือนจากเว็บ (Compatibility Warning)
  - ต้องการความโปร่งใสในสถานะขนส่ง เก็บไทม์ไลน์ไว้ดูย้อนหลังได้
* **ฟีเจอร์ ComHub ที่เข้ามาแก้ปัญหา (ComHub Mapping):**
  - **Compatibility Checker (C-02):** เตือนล่วงหน้าเมื่อเลือกอุปกรณ์เข้ากันไม่ได้ ช่วยลดโอกาสสั่งผิด
  - **Checkout with Slip Upload (SYS-03, SYS-04):** ระบบสั่งซื้อพร้อมอัปโหลดสลิปชำระเงิน (Mockup - บีบอัดเป็น WebP + Base64) เข้าใจง่าย
  - **Order Tracking (C-09):** ระบบแสดงสถานะ 5 ขั้นตอน (Pending Payment → Paid → Processing → Shipped → Delivered) พร้อมเลขพัสดุ ทำให้ลุงเห็นสถานะได้ตลอดเวลา

## 🧑‍💼 3. วิเคราะห์ผู้ใช้งานระบบฝั่งหลังบ้าน (Internal Users Persona)

เพื่อให้ระบบทำงานประสานกันได้สมบูรณ์แบบ แพลตฟอร์ม ComHub จึงต้องออกแบบมาให้ตอบโจทย์ผู้ใช้งานภายในด้วย โดยใน MVP มีเพียงบทบาทเดียวคือ **Admin** (ตัดบทบาท Staff และ Manager ออกจาก scope เดิม):

---

### Persona 4: ผู้ดูแลระบบ (Admin)

```
[ รูปภาพจำลอง: ชายหนุ่มนั่งหน้าจอหลายจอในห้องออฟฟิศ ]
ชื่อ: นายภูมิ (คุณภูมิ) | อายุ: 26 ปี
อาชีพ: เจ้าของร้านและผู้ดูแลระบบฐานข้อมูลสินค้าของ ComHub
ความคุ้นเคยไอที: ระดับสูงมาก (เป็นผู้ตั้งค่าระบบ, เข้าใจ RBAC และโครงสร้างฐานข้อมูล)
```

* **เป้าหมายและความต้องการ (Goals & Needs):**
  - ต้องการจัดการคลังสินค้าได้ครบ (เพิ่ม/แก้ไข/ปิดขาย) โดยไม่ต้องแก้ไข SQL โดยตรง
  - ต้องการตรวจสอบและอนุมัติสลิปการชำระเงินจากลูกค้าเพื่อปล่อยออเดอร์เข้าคิวจัดส่ง
  - ต้องการดูภาพรวมยอดขาย สินค้ายอดนิยม และเตือนสต็อกต่ำในหน้าเดียว
  - ต้องการสร้างและจัดการบัญชีผู้ดูแลระบบ (Admin) เพิ่มเติมผ่าน UI
* **ปัญหาและข้อจำกัด (Pain Points):**
  - กลัวว่าการแก้สต็อกโดยตรงในฐานข้อมูลจะทำให้ข้อมูลออเดอร์ที่กำลังดำเนินการอยู่ผิดพลาด (Data Conflict)
  - ต้องการ Soft Delete เพื่อซ่อนสินค้าที่เลิกขายโดยไม่ลบประวัติออเดอร์เก่าทิ้ง
  - ตรวจสลิปด้วยตาเปล่าเยอะๆ ทำให้พลาดง่าย ต้องการหน้ารวมคิวสลิปที่รอตรวจ
* **พฤติกรรมการใช้งาน (Behaviors):**
  - ใช้งานผ่านเดสก์ท็อปหรือโน้ตบุ๊ก ไม่เน้นมือถือ เพราะต้องดูรายละเอียดจำนวนมาก
  - ทำงานเป็น session: เปิดระบบหลังบ้านช่วงต้นวันเพื่ออนุมัติสลิปค้างคืน แล้วปิดได้เลย
  - ตรวจ Dashboard ทุกเย็นเพื่อเช็คสต็อกและยอดขาย
* **ฟีเจอร์ ComHub ที่เข้ามาแก้ปัญหา (ComHub Mapping):**
  - **Product Management CRUD (A-01):** เพิ่ม/แก้ไข/Soft Delete (is_active) สินค้า พร้อมอัปโหลดรูป WebP
  - **Payment Review (A-02):** คิวตรวจสลิปพร้อมภาพจริง (Base64 Mockup) กด Approve/Reject แล้วออเดอร์เปลี่ยนสถานะทันที
  - **Order Management (A-03):** ดู/อัปเดตสถานะออเดอร์ตาม 5 ขั้นตอน และกรอก Tracking Number
  - **Role & Access Control (A-04):** สร้างและจัดการบัญชี Admin (เฉพาะบทบาท Customer/Admin) พร้อม RBAC
  - **Dashboard & Reports (A-05):** ดูยอดขายรวม สินค้ายอดนิยม และเตือนสต็อกต่ำ (≤ 3 ชิ้น) ในหน้าเดียว
