# เอกสารวิเคราะห์กลุ่มผู้ใช้งานจำลอง (User Persona Analysis)

เอกสารนี้แสดงการวิเคราะห์กลุ่มผู้ใช้งานจำลอง (**User Personas**) ของระบบ **ComHub** เพื่อทำความเข้าใจพฤติกรรม เป้าหมาย ข้อจำกัด (Pain Points) และการออกแบบฟีเจอร์ของระบบให้ตอบโจทย์ผู้ใช้งานจริง ตามหลักการวิเคราะห์ระบบ (System Analysis) ในวิชา **csi 204**

---

## 🎯 รูปแบบโครงสร้างการวิเคราะห์ User Persona (Persona Template)
ในการจัดทำเล่มรายงาน ข้อมูลของ Persona แต่ละคนจะประกอบด้วยหัวข้อหลักดังนี้:
1. **ประวัติและข้อมูลพื้นฐาน (Demographics & Bio):** ชื่อ, อายุ, อาชีพ, และความคุ้นเคยกับเทคโนโลยี (Tech Savviness)
2. **เป้าหมายและความต้องการ (Goals & Needs):** สิ่งที่คาดหวังจะได้รับจากการเข้าใช้งานระบบ ComHub
3. **ปัญหาและข้อจำกัดในปัจจุบัน (Pain Points & Frustrations):** อุปสรรคที่เจอเมื่อใช้บริการเว็บจัดสเปคคอมฯ หรือเว็บไอทีอื่นๆ
4. **พฤติกรรมการใช้งาน (Behaviors):** ลักษณะการเลือกซื้อสินค้าและการใช้งานอุปกรณ์สื่อสาร
5. **ฟีเจอร์ของ ComHub ที่ตอบโจทย์ (ComHub Mapping):** ฟังก์ชันของระบบที่จะช่วยแก้ปัญหาให้เขาได้โดยตรง

---

## 👥 รายละเอียดกลุ่มผู้ใช้งานจำลอง (Detailed Personas)

### Persona 1: ลูกค้ากลุ่มวัยรุ่น / เกมเมอร์งบจำกัด (The Budget Gamer / Student)

```
[ รูปภาพจำลอง: เด็กหนุ่มวัยรุ่นสวมหูฟังเกมมิ่ง ]
ชื่อ: นายกฤษดา (น้องก้อง) | อายุ: 20 ปี
อาชีพ: นักศึกษาคณะวิศวกรรมศาสตร์
ความคุ้นเคยไอที: ระดับกลาง (ชอบเล่นเกม แต่ไม่เคยประกอบคอมพิวเตอร์ด้วยตัวเอง)
```

* **เป้าหมายและความต้องการ (Goals & Needs):**
  - ต้องการจัดสเปคคอมพิวเตอร์เล่นเกมที่มีงบจำกัดไม่เกิน 25,000 บาท
  - ต้องการเครื่องที่สามารถเล่นเกมระดับ AAA ในปัจจุบันได้ลื่นไหล
  - อยากลองจัดสเปคด้วยตนเองแทนการซื้อแบรนด์สำเร็จรูปเพื่อความคุ้มค่า
* **ปัญหาและข้อจำกัด (Pain Points):**
  - ไม่รู้วิธีการตรวจสอบว่าบอร์ดและ CPU ที่เลือกเข้ากันได้ไหม (กลัวซื้อมาแล้วใส่ไม่ได้)
  - ไม่รู้วิธีคำนวณกำลังไฟวัตต์ (TDP) ของ PSU กลัวเครื่องไฟดับหรือพังขณะเล่นเกม
  - มีงบน้อย กลัวโดนร้านค้าหลอกขายสินค้าที่ไม่คุ้มราคา
* **พฤติกรรม (Behaviors):**
  - ใช้สมาร์ทโฟนเป็นหลักในการดูข้อมูลสินค้า
  - ชอบอ่านรีวิวและดูรูปถ่ายคอมประกอบของคนอื่นเพื่อดูความสวยงามและประสิทธิภาพ
* **ฟีเจอร์ ComHub ที่เข้ามาแก้ปัญหา (ComHub Mapping):**
  - **PC Builder & Compatibility Logic:** ระบบกรอง Socket และขนาดการ์ดจอบังคับให้เลือกเฉพาะของที่เข้ากันได้
  - **Wattage Calculator:** มีระบบคำนวณ TDP รวมและกรองเฉพาะ PSU วัตต์ผ่านเกณฑ์ความปลอดภัย 1.2 เท่า
  - **PC Build Gallery:** หน้าคอมมูนิตี้ให้กดคัดลอกสเปคยอดนิยมในงบจำกัดที่แอดมินหรือผู้ใช้คนอื่นจัดไว้มาปรับแต่งต่อได้ทันที

### Persona 2: ลูกค้ากลุ่มคนทำงานสายครีเอทีฟ (The Professional Workstation Creator)

```
[ รูปภาพจำลอง: หญิงสาววัยทำงานนั่งหน้าจอคู่ในสตูดิโอ ]
ชื่อ: นางสาวมนัสวี (คุณเมย์) | อายุ: 28 ปี
อาชีพ: ดีไซเนอร์และนักตัดต่อวิดีโอฟรีแลนซ์
ความคุ้นเคยไอที: ระดับสูง (รู้สเปคคอมพิวเตอร์เป็นอย่างดี ต้องการอุปกรณ์เฉพาะทาง)
```

* **เป้าหมายและความต้องการ (Goals & Needs):**
  - ต้องการประกอบคอมพิวเตอร์สเปคสูง (Workstation) สำหรับตัดต่อวิดีโอ 4K และเรนเดอร์กราฟิก 3D
  - ต้องการเลือกการ์ดจอค่ายเขียวความจุสูง และเมนบอร์ดที่มีพอร์ตเชื่อมต่อครบถ้วน
  - ต้องการบริการประกอบเครื่องจากผู้เชี่ยวชาญของร้านและทดสอบความร้อนก่อนส่งมอบ
* **ปัญหาและข้อจำกัด (Pain Points):**
  - การ์ดจอตัดต่อมักมีขนาดใหญ่และยาวมาก กลัวเลือกเคสมาแล้วใส่การ์ดจอลงไปไม่ได้
  - ทำงานฟรีแลนซ์ไม่มีเวลามานั่งประกอบและลงโปรแกรมเทสอุณหภูมิเอง
  - กลัวว่าคอมพิวเตอร์ที่จัดประกอบไปแล้วจะไม่เสถียร (System Crash) ระหว่างเรนเดอร์งานส่งลูกค้า
* **พฤติกรรม (Behaviors):**
  - ใช้งานคอมพิวเตอร์เดสก์ท็อปเป็นหลักในการเลือกซื้อ
  - ชอบเปรียบเทียบสเปคทางเทคนิคเชิงลึก (เช่น แบนด์วิดท์, บัสแรม, กำลังไฟ)
* **ฟีเจอร์ ComHub ที่เข้ามาแก้ปัญหา (ComHub Mapping):**
  - **GPU Length Compatibility Check:** บังคับเช็คขนาดการ์ดจอกับพื้นที่ความกว้างสูงสุดที่เคสรองรับ
  - **Product Comparison Table:** หน้าเปรียบเทียบสเปคชิ้นส่วนเทียบได้สูงสุด 3 อุปกรณ์
  - **Assembly & Burn-in Tracking:** หน้า UAT Tracking แสดงผลการทดสอบ Burn-in (อุณหภูมิ CPU/GPU) บันทึกโดยพนักงานประกอบ เพื่อให้ลูกค้ามั่นใจเรื่องความร้อนก่อนจัดส่ง

### Persona 3: ลูกค้าทั่วไป / ผู้ปกครองจัดคอมฯ ให้ลูก (The Casual Buyer / Parent)

```
[ รูปภาพจำลอง: ชายวัยกลางคนยืนยิ้มในร้านกาแฟ ]
ชื่อ: นายสมเกียรติ (ลุงสมเกียรติ) | อายุ: 52 ปี
อาชีพ: เจ้าของธุรกิจส่วนตัว (ร้านอาหาร)
ความคุ้นเคยไอที: ระดับต่ำ (ใช้งานสมาร์ทโฟนทั่วไปได้ ไม่มีความรู้เรื่องสเปคฮาร์ดแวร์เลย)
```

* **เป้าหมายและความต้องการ (Goals & Needs):**
  - ต้องการซื้อคอมพิวเตอร์ให้ลูกสาวใช้เรียนออนไลน์และทำงานส่งครูในงบกลางๆ
  - ต้องการความสะดวกในการสั่งซื้อ ไม่ต้องเลือกอุปกรณ์ทีละชิ้นเอง
  - ต้องการบริการจัดส่งถึงบ้านแบบประกอบเสร็จพร้อมใช้งานทันที (Plug and Play)
* **ปัญหาและข้อจำกัด (Pain Points):**
  - สับสนกับศัพท์เทคนิคไอที (เช่น LGA1700, DDR5, PCIe, NVMe, TDP)
  - กลัวสั่งไปแล้วเปิดไม่ติด หรือต้องมานั่งประกอบสายไฟเองซึ่งทำไม่เป็น
  - กลัวโดนโกงเงิน และอยากรู้สถานะของเครื่องว่าประกอบไปถึงขั้นตอนไหนแล้ว
* **พฤติกรรม (Behaviors):**
  - ซื้อของออนไลน์ผ่านสมาร์ทโฟน
  - เชื่อถือคำแนะนำของร้านค้า และต้องการความโปร่งใสในสถานะขนส่ง
* **ฟีเจอร์ ComHub ที่เข้ามาแก้ปัญหา (ComHub Mapping):**
  - **Pre-built Templates:** สามารถกดสั่งซื้อเซ็ตสเปคแนะนำของร้านค้าที่จัดเตรียมไว้ให้ตามงบประมาณได้ในคลิกเดียว (ไม่ต้องเลือกอุปกรณ์เอง)
  - **Checkout with Slip Upload:** ระบบสั่งซื้อพร้อมอัปโหลดสลิปชำระเงินที่เข้าใจง่าย ปลอดภัย
  - **Order Assembly Tracking:** ระบบแสดงสถานะการประกอบเครื่องให้ลุงเห็นแบบสดๆ 4 ขั้นตอน ทำให้ลุงอุ่นใจว่าเงินที่จ่ายไปได้รับการดำเนินการจริง

## 🧑‍💼 3. วิเคราะห์ผู้ใช้งานระบบฝั่งหลังบ้าน (Internal Users Persona)

เพื่อให้ระบบทำงานประสานกันได้สมบูรณ์แบบ แพลตฟอร์ม ComHub จึงต้องออกแบบมาให้ตอบโจทย์ผู้ใช้งานภายใน (Staff, Manager) ด้วยเช่นกัน:

1. **พนักงานประกอบคอมฯ (Staff - พี่เอ็ม):**
   - *พฤติกรรม & ความต้องการ:* ต้องการระบบคิวงานประกอบคอมฯ ที่ดูง่าย มีใบรายการอุปกรณ์ (BOM - Bill of Materials) บอกครบถ้วนว่าต้องไปหยิบชิ้นส่วนไหนในสต็อกมาจัดประกอบบ้าง และมีฟอร์มสั้นๆ ให้กรอกอุณหภูมิที่เทสผ่านได้รวดเร็วผ่านแท็บเล็ตขณะทำงานในห้องเวิร์กช็อป
2. **ผู้จัดการร้าน (Manager - คุณเดชา):**
   - *พฤติกรรม & ความต้องการ:* ต้องการข้อมูลแดชบอร์ดสรุปยอดขายบนมือถือเพื่อดูสถานการณ์ค้า และระบบคัดกรองรูปถ่ายรีวิวจากหน้าเว็บบอร์ด เพื่อกดยืนยันให้ขึ้นสาธารณะได้รวดเร็ว ป้องกันภาพขยะหรือสลิปปลอม
