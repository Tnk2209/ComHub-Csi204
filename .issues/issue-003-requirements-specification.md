# Issue 003: SDLC Phase 2 (Analysis) - Functional & Non-Functional Requirements Definition

## What to build
จัดทำเอกสารความต้องการของระบบ (System Requirement Specification - SRS) โดยระบุ Functional Requirements (FR) และ Non-Functional Requirements (NFR) อย่างเป็นระบบ

## Acceptance criteria
- [ ] สรุป Functional Requirements ของสิทธิ์ Customer (C-01 ถึง C-09)
- [ ] สรุป Functional Requirements ของสิทธิ์ Staff (S-01 ถึง S-03)
- [ ] สรุป Functional Requirements ของสิทธิ์ Manager (M-01 ถึง M-03)
- [ ] สรุป Functional Requirements ของสิทธิ์ Admin (A-01 ถึง A-04)
- [ ] กำหนด Non-Functional Requirements ในประเด็น: ประสิทธิภาพการโหลด (Sub-500ms), ความปลอดภัยของข้อมูลคำสั่งซื้อและการแยกสิทธิ์ (RBAC), ความน่าเชื่อถือของสต็อก และมาตรฐาน UI/UX
