1. ชื่อโครงงาน (Project Title)
 - ภาษาไทย: แพลตฟอร์มอีคอมเมิร์ซสำหรับจัดสเปคและจำหน่ายอุปกรณ์คอมพิวเตอร์ครบวงจร
 - ภาษาอังกฤษ: BuildIT: Comprehensive E-Commerce Platform for Custom PC Building and IT Accessories
 - ภาษาไทย: คอมฮับ
 - ภาษาอังกฤษ : ComHub
 
2. หลักการและเหตุผล (Rationale)
 ในปัจจุบันความต้องการใช้งานคอมพิวเตอร์ประสิทธิภาพสูง ทั้งสำหรับกลุ่มเกมเมอร์ (Gaming) และคนทำงาน (Work from Home / Creators) มีอัตราการเติบโตอย่างต่อเนื่อง แต่ปัญหาหลักของผู้ใช้งานหลายคนคือ "การจัดสเปคคอมพิวเตอร์ให้ชิ้นส่วนทำงานร่วมกันได้" 
และความยุ่งยากในการเปรียบเทียบราคา โครงงานนี้จึงมุ่งเน้นพัฒนาเว็บไซต์ที่รวบรวมอุปกรณ์คอมพิวเตอร์ไว้ในที่เดียว พร้อมฟีเจอร์ช่วยอำนวยความสะดวกในการเลือกซื้อสินค้า และระบบจัดการหลังบ้านที่มีประสิทธิภาพ เพื่อให้การบริหารจัดการของทีมงาน (Staff, Manager, Admin) เป็นระบบและตรวจสอบได้ง่ายขึ้น

3. วัตถุประสงค์ของโครงงาน (Objective)
 เพื่อพัฒนาระบบพาณิชย์อิเล็กทรอนิกส์ (E-Commerce) ที่รองรับการซื้อขายอุปกรณ์คอมพิวเตอร์ และมีระบบหลังบ้านที่แบ่งแยกสิทธิ์การทำงานอย่างชัดเจน
แบ่งเป็นข้อ
	- เพื่อสร้างเว็บไซต์ที่ลูกค้าสามารถเลือกซื้ออุปกรณ์เกมมิ่งและอุปกรณ์ทำงานได้อย่างสะดวก
	- เพื่อพัฒนาระบบ "จัดสเปคคอมพิวเตอร์" (Custom PC Builder) ที่ช่วยให้ลูกค้าเลือกชิ้นส่วนได้ง่ายขึ้น
	- เพื่อพัฒนาระบบจัดการหลังบ้าน (Back-Office) ที่รองรับการทำงานของพนักงานหลากหลายระดับ
	- เพื่อศึกษาและประยุกต์ใช้เทคโนโลยีการพัฒนาเว็บสมัยใหม่ (React, Node.js) ในการทำงานจริง

4. ขอบเขตของระบบ (System Scope)
 - ลูกค้า (Customer): เลือกดูสินค้า จัดสเปคคอมพิวเตอร์ สั่งซื้อสินค้า และดูประวัติการสั่งซื้อของตนเอง
 - พนักงาน (Staff): รับออเดอร์ อัปเดตสถานะการจัดส่ง และตอบคำถามลูกค้า
 - ผู้จัดการ (Manager): ดูสรุปยอดขาย (Dashboard) จัดการโปรโมชั่น และดูรายงานภาพรวม
 - ผู้ดูแลระบบ (Administrator): จัดการข้อมูลพนักงาน กำหนดสิทธิ์ผู้ใช้งาน และจัดการข้อมูลพื้นฐานของระบบ (เช่น เพิ่ม/ลดหมวดหมู่สินค้า)
ความสามารถหลักของระบบ (Main Functions):
 1. ระบบแคตตาล็อกและค้นหาสินค้า: ค้นหา กรองสินค้าตามหมวดหมู่ (เช่น CPU, GPU, แป้นพิมพ์) และดูรายละเอียดเชิงลึกได้
 2. ระบบจัดสเปคคอม (PC Builder): หน้าต่างพิเศษที่ให้ลูกค้าเลือกชิ้นส่วนทีละหมวดหมู่ พร้อมคำนวณราคารวมให้อัตโนมัติ
 3. ระบบตะกร้าสินค้าและการชำระเงิน: จัดการสินค้าในตะกร้า และจำลองกระบวนการ Checkout (อัปโหลดสลิป หรือกรอกที่อยู่)
 4. ระบบจัดการคำสั่งซื้อ (Order Management): ให้พนักงาน (Staff) เข้ามาเปลี่ยนสถานะ (เช่น รอชำระเงิน -> กำลังจัดส่ง -> สำเร็จ)
 5. ระบบจัดการสินค้า (Product Management): ให้ Admin/Manager เข้ามา เพิ่ม ลบ แก้ไข ข้อมูลสินค้าและสต็อก
 
5. แนวทางการพัฒนาตาม SDLC
 - Planning: กำหนดขอบเขตของโปรเจ็ค แบ่งหน้าที่ความรับผิดชอบ และวางแผนตารางเวลา 4 สัปดาห์
 - Analysis: รวบรวมความต้องการของระบบ ออกแบบ User Flow ว่าผู้ใช้งานแต่ละระดับจะเข้าถึงหน้าไหนได้บ้าง
 - Design: ออกแบบหน้าจอ (UI/UX) ด้วยเครื่องมืออย่าง Figma และออกแบบโครงสร้างข้อมูล (Data Structure)
 - Development: เริ่มเขียนโค้ด ฝั่งหน้าบ้าน (FrontEnd) และหลังบ้าน (BackEnd) ให้เชื่อมต่อกัน
 - Testing: ทำการทดสอบระบบ ค้นหาข้อผิดพลาด (Bugs) และตรวจสอบว่าฟังก์ชั่นทำงานตรงตามที่ออกแบบไว้
 - Deployment: นำเว็บไซต์ขึ้นโฮสต์จำลอง (เช่น Vercel, Render) เพื่อให้อาจารย์และเพื่อนๆ สามารถเข้าทดสอบได้
 - Maintenance: เตรียมพร้อมแก้ไขโค้ดหากพบปัญหาระหว่างการนำเสนอ หรือนำข้อเสนอแนะจากอาจารย์มาปรับปรุง
 
6. เครื่องมือ และ เทคโนโลยีที่ใช้ (Tools & Technologies)
 - FrontEnd: React + Vite (เพื่อความรวดเร็วในการ Build) + Tailwind CSS (สำหรับการจัดการ UI อย่างสวยงามและเป็นระเบียบ)
 - BackEnd: Node.js + Express (TypeScript) เพื่อความปลอดภัยของชนิดข้อมูล (Type Safety)
 - Database: LocalStorage : ใช้จำลองการเก็บ Session หรือ Cart ก่อนส่งไป BackEnd ได้ครับ!
 
7. แนวทางการทดสอบ
 - ประเภทการทดสอบ: User Acceptance Testing (UAT) (การทดสอบการยอมรับของผู้ใช้)
 - เครื่องมือที่ใช้: Manual เทสด้วยตนเอง
 - วิธีการ: สร้าง Test Case หรือจำลองสถานการณ์จริง
 
8. ผลที่คาดว่าจะได้รับ (Expected Outcomes)
 - ได้เว็บไซต์ E-Commerce ที่สามารถจำลองการซื้อขายอุปกรณ์คอมพิวเตอร์ได้จริง
 - ลดความซับซ้อนในการเลือกซื้อชิ้นส่วนคอมพิวเตอร์ผ่านระบบ PC Builder
 - มีระบบหลังบ้านที่ช่วยให้การบริหารจัดการข้อมูลเป็นระบบ และป้องกันการเข้าถึงข้อมูลผิดพลาดผ่านการจำกัดสิทธิ์ (Role-based)
 - ได้รับประสบการณ์การทำงานจริงแบบ Full-Stack Development และการประยุกต์ใช้
 - ได้ผลงาน (Portfolio) ที่สามารถนำไปต่อยอดในการสมัครงานในสาย Software Engineer หรือ Web Developer ได้ในอนาคต
 
9. แผนการดำเนินงาน ตลอด 4 สัปดาห์
สัปดาห์ที่,ขั้นตอนการดำเนินงาน,รายละเอียดงาน (Task)
1,วิเคราะห์และออกแบบระบบ,"สรุป Requirement, ร่าง Wireframe/UI, ออกแบบโครงสร้างข้อมูล JSON, แบ่งงานกันในทีม"
2,พัฒนา FrontEnd,"สร้าง Component ต่างๆ ด้วย React, จัด Layout ด้วย Tailwind, ทำหน้า Home, Shop, PC Builder"
3,พัฒนา BackEnd,"สร้าง API ด้วย Node.js/Express, ทำระบบ Login จำลอง, เชื่อมต่อ API กับ FrontEnd"
4,ทดสอบและนำเสนอ,"ทำ UAT ตาม Flow การใช้งาน, แก้ไข Bugs, เตรียมสไลด์และฝึกซ้อมนำเสนอผลงาน"
